What Causes a Slow Shopify Store?

  • Unoptimized images: JPEG/PNG images above 200 KB are the single biggest drag on load time. Modern formats like WebP and AVIF cut file sizes by 50–80%.
  • Too many third-party apps: Every Shopify app you install adds JavaScript to your storefront. Even apps you rarely use slow down every page load.
  • Render-blocking scripts: Scripts that load synchronously in the <head> block the browser from rendering your page until they finish downloading.
  • Unoptimized Liquid code: Inefficient loops, redundant API calls, and heavy snippet includes inflate server response times.
  • No lazy loading: Loading all images at once wastes bandwidth for content the user may never scroll to.
  • Font loading issues: Loading multiple font weights without font-display: swap causes invisible text during load (FOIT).

Core Web Vitals Explained

Google uses three Core Web Vitals metrics to measure real-world user experience. Understanding them helps you see why certain fixes matter more than others.

MetricWhat It MeasuresGood ThresholdPoor Threshold
LCP — Largest Contentful PaintHow fast your main content loadsUnder 2.5sAbove 4.0s
CLS — Cumulative Layout ShiftHow much the page jumps aroundUnder 0.1Above 0.25
FID/INP — Interaction to Next PaintHow fast your page responds to clicksUnder 200msAbove 500ms
Pro Tip: CLS is often caused by images without explicit width/height attributes and third-party widgets that inject content after the page loads. Fixing CLS alone can improve your Google ranking without touching a single line of business logic.
