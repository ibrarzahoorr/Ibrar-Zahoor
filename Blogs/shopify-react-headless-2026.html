What is Headless Shopify Commerce?

In a traditional Shopify store, the frontend (what customers see) is tightly coupled to the Shopify Liquid theme system. In headless commerce, the frontend is completely decoupled — you build it with React, Next.js, Vue, or any framework you choose — and it communicates with Shopify exclusively through APIs.

Ibrar Zahoor describes it as: "Shopify's commerce brain — products, inventory, orders, checkout — powering a completely custom user experience."

Shopify Hydrogen: The Official Headless Framework

Hydrogen is Shopify's official React framework for headless commerce, built on Remix. It comes with first-party Shopify integrations built in, making it the fastest path to a production headless Shopify store for React teams.

What Hydrogen Provides

  • Storefront API client: Pre-configured GraphQL client for querying Shopify data
  • Cart hooks: useCart(), CartProvider — optimistic cart updates out of the box
  • Image optimization: Shopify CDN image transformations with responsive images
  • Performance defaults: Route-based code splitting, streaming SSR via Remix
  • Oxygen deployment: Shopify's edge-deployed hosting for Hydrogen apps (included in Shopify plans)
  • Shopify Analytics: Pixel and analytics events for Shopify reporting

Getting Started with Hydrogen

npm create @shopify/hydrogen@latest

This scaffolds a new Hydrogen project with a demo store template. You configure your Shopify Storefront API credentials in the .env file and can immediately run a headless Shopify storefront locally.

Custom React + Next.js Headless Shopify

The alternative to Hydrogen is building a custom headless Shopify storefront using Next.js and the Shopify Storefront API directly. This approach gives you maximum control and is favored by teams already deeply experienced in the Next.js ecosystem.

Architecture: Next.js + Shopify Storefront API

  1. Data fetching: Use Next.js getStaticProps / getServerSideProps (App Router: Server Components) to fetch product data from Storefront API at build time or request time.
  2. Cart management: Use Shopify's cart mutations (cartCreate, cartLinesAdd) and store cart ID in a cookie. Render cart state client-side with React state or Zustand/Context.
  3. Checkout: Redirect to Shopify-hosted checkout via the cart's checkoutUrl or use custom checkout extensions for checkout customization.
  4. Hosting: Deploy on Vercel (recommended for Next.js) with edge functions for optimal global performance.
Ibrar Zahoor's Preferred Stack for Headless Shopify in 2026: Next.js 15 (App Router) + Shopify Storefront API (GraphQL) + TypeScript + TailwindCSS + Vercel hosting. This stack delivers sub-second page loads, full SEO compatibility via Server Components, and a developer experience that's hard to beat.

Shopify Storefront API: The Core of Headless

The Storefront API is a public GraphQL API that exposes all the data your frontend needs to render a shopping experience. Key queries and mutations:

Essential Storefront API Queries

// Fetch a product by handle
query GetProduct($handle: String!) {
  product(handle: $handle) {
    id
    title
    descriptionHtml
    priceRange { minVariantPrice { amount currencyCode } }
    images(first: 10) { edges { node { url altText } } }
    variants(first: 100) {
      edges { node { id title price { amount } availableForSale } }
    }
  }
}

Cart Mutations

  • cartCreate — Create a new cart with optional lines and attributes
  • cartLinesAdd — Add items to an existing cart
  • cartLinesUpdate — Update quantity of cart lines
  • cartLinesRemove — Remove items from cart
  • cartDiscountCodesUpdate — Apply discount codes

SEO in Headless Shopify: What You MUST Get Right

Headless commerce can be catastrophic for SEO if implemented incorrectly (client-side-only React SPAs are essentially invisible to Google crawlers). The non-negotiable requirements:

  • Server-Side Rendering (SSR) or Static Site Generation (SSG): Every product, collection, and page must render its HTML on the server. React Server Components in Next.js 15 make this natural.
  • Canonical tags: Implement canonical tags manually — Shopify's automatic canonical injection doesn't apply to headless storefronts.
  • Sitemap generation: Generate a sitemap.xml by querying all products and collections from Shopify and outputting a Next.js/Hydrogen sitemap route.
  • Schema markup: All Product, BreadcrumbList, and FAQPage schema must be implemented in the headless frontend.
  • Hreflang (if multilingual): Must be rendered server-side for each language/region.

Hydrogen vs Custom Next.js: The Decision Framework

Choose Hydrogen if:

  • Your team knows Remix
  • You want Shopify's official support and maintained integrations
  • You want to deploy on Oxygen (Shopify's edge hosting, included free)
  • You're building a standard DTC store headless

Choose Next.js + Storefront API if:

  • Your team has deep Next.js/React expertise
  • You need to integrate multiple data sources (CMS, ERP, other APIs)
  • You want to deploy on Vercel or your own infrastructure
  • You're building a complex multi-region or marketplace experience
  • You need more control than Hydrogen's opinionated structure allows

When NOT to Go Headless

Headless is not always the right answer. Ibrar Zahoor advises against headless Shopify when:

  • Budget is under $15,000 (traditional theme delivers better ROI)
  • The team has no React/Next.js experience
  • Time to launch is under 4 weeks
  • The store uses many Shopify apps that inject frontend code (compatibility issues are common)
  • The performance gap vs. a well-optimized Liquid theme doesn't justify the cost
