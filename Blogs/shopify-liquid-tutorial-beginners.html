Shopify Liquid Tutorial for Beginners – Objects, Tags & Filters

Master Shopify's templating language from the ground up — learn objects, tags, filters, loops, conditionals, and write your first real Liquid template.

January 20, 2026 14 min read Ibrar Zahoor

If you want to customize Shopify themes beyond what the drag-and-drop editor allows, you need to understand Shopify Liquid. Liquid is an open-source templating language created by Shopify — it is the engine behind every Shopify store's HTML output. This tutorial for beginners covers everything: what Liquid is, how it works, its three core building blocks, practical examples, and how to write your first complete product template.

What Is Shopify Liquid?

Liquid is a safe, customer-facing template language. Unlike PHP or JavaScript, Liquid code runs on Shopify's servers before the page is sent to the browser — the browser only ever receives clean HTML. This makes it fast and secure. Every .liquid file in a Shopify theme is a Liquid template.

Liquid files live inside theme folders like templates/, sections/, snippets/, and layout/. When a customer visits a product page, Shopify finds the matching template, processes all the Liquid code, and serves the resulting HTML.

The Three Building Blocks of Liquid

Building BlockSyntaxPurpose
Objects{{ }} double curly bracesOutput dynamic values
Tags{% %} curly brace percentLogic, loops, includes
Filters| filter_name pipe notationTransform output values

Liquid Objects – Outputting Dynamic Data

Objects are the most fundamental part of Liquid. They output content from your Shopify store into the page. You write an object by wrapping it in double curly braces.

{{ product.title }}
{{ product.price | money }}
{{ shop.name }}
{{ customer.first_name }}

Shopify provides dozens of global objects. The most commonly used ones are:

  • product — the current product (title, description, variants, images, price)
  • collection — the current collection and its products
  • cart — the current cart (items, total, item count)
  • customer — the logged-in customer's data
  • shop — your store's name, email, domain, currency
  • request — current URL, page type, locale
Pro Tip: In the Shopify theme editor, open any template and use {{ product | json }} temporarily to dump the full object as JSON — this reveals every available property on that object.

Liquid Tags – Controlling Logic and Flow

Tags use the {% %} syntax and do not produce visible output themselves — they control what gets rendered. There are three categories of tags: control flow, iteration, and theme tags.

Control Flow Tags (if / unless / elsif / else)

{% if product.available %}
  <button>Add to Cart</button>
{% elsif product.available == false %}
  <button disabled>Sold Out</button>
{% endif %}

{% unless customer %}
  <a href="/account/login">Log in</a>
{% endunless %}

Iteration Tags (for loops)

{% for product in collection.products %}
  <div class="product-card">
    <h2>{{ product.title }}</h2>
    <p>{{ product.price | money }}</p>
  </div>
{% endfor %}

{% comment %}Limit to first 4 products{% endcomment %}
{% for product in collection.products limit:4 %}
  {{ product.title }}
{% endfor %}

Variable Tags (assign / capture)

{% assign discount_price = product.price | times: 0.9 %}
{{ discount_price | money }}

{% capture full_name %}{{ customer.first_name }} {{ customer.last_name }}{% endcapture %}
{{ full_name }}

Theme Tags (render / include)

{% render 'product-card', product: product %}
{% render 'icon-search' %}

Liquid Filters – Transforming Output

Filters modify the output of objects. They are chained with the pipe character | and can be stacked in sequence.

FilterExampleOutput
upcase{{ "hello" | upcase }}HELLO
downcase{{ "HELLO" | downcase }}hello
capitalize{{ "ibrar zahoor" | capitalize }}Ibrar zahoor
money{{ product.price | money }}$29.99
date{{ "now" | date: "%B %d, %Y" }}April 30, 2026
truncate{{ product.description | truncate: 80 }}First 80 chars...
strip_html{{ product.description | strip_html }}Plain text only
size{{ cart.items | size }}Number of items
default{{ var | default: "N/A" }}"N/A" if var is blank
url_encode{{ "hello world" | url_encode }}hello+world

Writing Your First Product Template

Here is a minimal but complete product template that combines objects, tags, and filters. In Dawn and OS 2.0 themes, product templates live in templates/product.json (JSON template) and corresponding sections. For older themes, create templates/product.liquid.

<div class="product">
  <h1>{{ product.title }}</h1>

  {% if product.featured_image %}
    <img src="{{ product.featured_image | img_url: '800x' }}"
         alt="{{ product.featured_image.alt | escape }}"
         width="800" loading="lazy">
  {% endif %}

  <div class="price">
    {% if product.compare_at_price > product.price %}
      <s>{{ product.compare_at_price | money }}</s>
    {% endif %}
    <strong>{{ product.price | money }}</strong>
  </div>

  <div class="description">
    {{ product.description }}
  </div>

  {% form 'product', product %}
    <select name="id">
      {% for variant in product.variants %}
        <option value="{{ variant.id }}"
          {% unless variant.available %}disabled{% endunless %}>
          {{ variant.title }} – {{ variant.price | money }}
        </option>
      {% endfor %}
    </select>
    <button type="submit">
      {% if product.available %}Add to Cart{% else %}Sold Out{% endif %}
    </button>
  {% endform %}
</div>

Liquid in Sections and Schemas

In Online Store 2.0 themes, sections use a {% schema %} tag at the bottom to define merchant-editable settings. These settings become accessible as the section.settings object in Liquid.

{{ section.settings.heading }}

{% schema %}
{
  "name": "Custom Banner",
  "settings": [
    {
      "type": "text",
      "id": "heading",
      "label": "Banner Heading",
      "default": "Welcome to our store"
    }
  ]
}
{% endschema %}
Pro Tip: Always use the | escape filter on user-generated content (like customer.name or URL parameters) to prevent XSS injection in your templates.

Common Beginner Mistakes in Liquid

  • Forgetting to close tags — every {% if %} needs a {% endif %}, every {% for %} needs an {% endfor %}.
  • Using include instead of render — {% include %} is deprecated in OS 2.0 themes. Use {% render %} instead.
  • Not using the money filter on prices — Shopify stores prices in cents. {{ product.price }} outputs 2999 not $29.99. Always pipe through | money.
  • Nested for loops without forloop.index — use {{ forloop.index }} and {{ forloop.index0 }} to track position when you need it.
