Shopify Admin API: The Backbone of App Development

The Admin API is the most feature-rich Shopify API. It gives programmatic access to nearly everything in a merchant's store: products, orders, customers, inventory, discounts, shipping, metafields, webhooks, and more.

Admin API Authentication

There are two paths to an Admin API access token:

  1. OAuth Flow (Public Apps): Used for apps distributed on the Shopify App Store. The merchant installs the app and grants scopes; Shopify issues an offline access token.
  2. Custom App Tokens: Used for private integrations. Go to Settings → Apps → Develop apps, create an app, configure API scopes, then install to generate a token. This token is shown once — store it securely.
Security Rule by Ibrar Zahoor: Never expose Admin API access tokens in client-side JavaScript, public repositories, or environment files committed to version control. Use server-side environment variables (e.g., in Node.js: process.env.SHOPIFY_ACCESS_TOKEN) and rotate tokens if compromised.

Admin API Scopes

Access tokens are scope-limited. Request only the scopes your integration needs — this is both a security best practice and a requirement for Shopify App Store approval. Common scopes include:

  • read_products, write_products
  • read_orders, write_orders
  • read_customers, write_customers
  • read_inventory, write_inventory
  • read_metaobjects, write_metaobjects

REST Admin API vs GraphQL Admin API

Shopify supports both REST and GraphQL Admin APIs, but the direction of travel is clear: Shopify is progressively deprecating REST endpoints in favor of GraphQL. As of 2026, several REST resources (Collections, Webhooks, Events) are deprecated or removed.

Shopify Storefront API: Power Behind Headless Commerce

The Storefront API is designed for building custom shopping experiences — headless React/Next.js storefronts, mobile apps, IoT kiosks, or any frontend that isn't a standard Shopify theme. It exposes only customer-safe data and actions.

What the Storefront API Exposes

  • Product and collection browsing
  • Cart creation, update, and discount application
  • Checkout creation and completion
  • Customer account creation and authentication
  • Metafield reads (public)
  • Predictive search

Storefront API Authentication

The Storefront API uses a public storefront access token — it can safely be embedded in client-side code. Generate it under Settings → Apps → Develop apps → Storefront API. Note that the token grants read access by default; write access (cart mutations) is always allowed, but customer write scopes must be explicitly enabled.

Shopify Ajax API: For Theme Development

The Ajax API is a lighter, browser-native API available within Shopify themes without any authentication. It's what Liquid themes use to add to cart, update cart, fetch product JSON, and more.

  • POST /cart/add.js — Add items to cart
  • GET /cart.js — Fetch current cart state
  • POST /cart/update.js — Update item quantities
  • GET /products/[handle].js — Fetch product JSON

The Ajax API is ideal for cart drawer implementations and product variant switchers without full page reloads. Ibrar Zahoor uses it extensively in custom Shopify theme builds.

Bulk Operations for Large Data Sets

When you need to fetch thousands of products, orders, or customers, use Shopify's GraphQL Bulk Operations. Instead of paginating through thousands of pages, you trigger a bulkOperationRunQuery mutation, and Shopify processes it asynchronously, delivering a JSONL file URL when done. This bypasses rate limits for large data exports.

Webhooks: React to Store Events

Webhooks let your application respond in real time to events in a Shopify store without polling. Common webhook topics include:

  • orders/create — Fire when a new order is placed
  • orders/paid — Fire when payment is captured
  • products/update — Fire when product data changes
  • inventory_levels/update — Fire when stock changes
  • customers/create — Fire when a new customer registers

In 2026, Shopify supports HTTP webhooks and PubSub/EventBridge for high-volume stores. Always verify webhook HMAC signatures to prevent spoofed requests.

Shopify API Versioning

Shopify releases new API versions quarterly (e.g., 2026-01, 2026-04, 2026-07). Each version is supported for 12 months before deprecation. Always pin your integration to a specific version and review release notes with each quarterly update. Ibrar Zahoor schedules quarterly API version reviews for all maintained client projects.

Shopify API Selection Guide (Quick Reference)

  • Building a Shopify app: GraphQL Admin API + Webhooks
  • Building a headless storefront: Storefront API (GraphQL)
  • Custom theme cart interactions: Ajax API
  • ERP/WMS sync: GraphQL Admin API (Bulk Operations for large data)
  • Mobile app: Storefront API
  • Legacy REST integration: Migrate to GraphQL ASAP
