Shopify REST API vs GraphQL API

Shopify offers two primary API paradigms: a RESTful Admin API and a GraphQL Admin API. Choosing the right one for your integration isn't a preference — it's an architectural decision that affects performance, maintainability, and rate limit efficiency.

The REST API is request-based and resource-oriented. You call /admin/api/2024-01/orders.json and get an array of orders. It's familiar, well-documented, and ideal for simple CRUD operations: creating products, fetching orders, updating inventory levels. The downside is over-fetching — every response returns the full resource object whether you need 3 fields or 30.

GraphQL: Request Exactly What You Need

GraphQL lets you specify precisely which fields to retrieve in a single query. A product query can fetch title, variants, and metafields in one round trip — no chaining REST calls. More importantly, Shopify's GraphQL API supports bulk operations via JSONL, allowing you to export millions of records asynchronously without hitting rate limits — something the REST API simply cannot do.

Common Integration Use Cases

The most impactful Shopify integrations fall into four categories. Each has specific data flows, sync frequencies, and error handling requirements that determine the right architecture.

ERP Integration — SAP, NetSuite, Odoo

Enterprise Resource Planning systems need bidirectional sync: Shopify sends orders to the ERP for fulfillment and financial recording; the ERP sends inventory levels and product updates back to Shopify. This requires a middleware layer with transformation logic, conflict resolution, and idempotent writes to prevent duplicate records.

CRM Integration — HubSpot, Salesforce

Every Shopify customer event — new registration, first purchase, repeat purchase, refund — should flow into your CRM to build a complete customer profile. Shopify's Customer and Order APIs feed into CRM contact records, deal pipelines, and lifecycle stage automation. Webhook-driven integrations ensure sub-second latency for real-time CRM updates.

3PL Integration — ShipBob, ShipStation, Easypost

Third-party logistics integrations push new orders from Shopify to your 3PL's system within seconds of placement. The 3PL returns tracking numbers and fulfillment events, which the integration writes back to Shopify using the Fulfillment API — automatically triggering branded shipping notification emails to customers.

Accounting — Xero, QuickBooks

Financial integrations map Shopify orders, refunds, and payouts to accounting journal entries. Every payment gateway has different payout timing and fee structures. A well-built accounting integration handles multi-currency, tax jurisdictions, and reconciliation between Shopify Payments deposits and your Xero/QuickBooks chart of accounts.

Architecture Tip: Never integrate Shopify directly with a downstream system in a point-to-point fashion. Always use a message queue (AWS SQS, Google Pub/Sub, or Redis) as an event buffer. This decouples the systems, handles burst traffic without data loss, and makes each component independently deployable.

Webhooks — Event-Driven Automation

Polling the Shopify API every few minutes is wasteful and slow. Webhooks push events to your endpoint the moment they happen — making your integration truly real-time. Shopify supports over 50 webhook topics covering the full commerce lifecycle.

Essential Shopify Webhook Topics

  • orders/create — Fires when a new order is placed; trigger fulfillment pipeline
  • orders/updated — Fires on any order change: status, address, line items
  • fulfillments/create — Fire when a 3PL confirms shipment; push tracking to customer
  • inventory_levels/update — Real-time stock level changes across all locations
  • customers/create & update — Feed new and updated profiles to CRM
  • refunds/create — Trigger accounting entries and loyalty point adjustments
  • checkouts/create — Start abandoned cart recovery timer in Klaviyo

All incoming webhooks must be verified using the HMAC-SHA256 signature in the X-Shopify-Hmac-Sha256 header. Your endpoint should return HTTP 200 within 5 seconds — any longer and Shopify will retry up to 19 times over 48 hours before marking the webhook delivery as failed. For processing-intensive tasks, acknowledge the webhook immediately and hand off to an async worker queue.

Frequently Asked Questions

What are Shopify API rate limits? +
Shopify REST API allows 40 requests per app per store per second using a leaky bucket algorithm. GraphQL uses a cost-based limit of 1,000 points per second. Custom integrations should implement retry logic with exponential backoff and leverage GraphQL bulk operations for large data sets to avoid hitting these limits.
How do you secure Shopify API integrations? +
All webhooks are verified using HMAC-SHA256 signature validation against the shared secret. OAuth 2.0 handles app authentication for multi-store scenarios. API credentials are stored in encrypted environment variables — never in client-side code or version control. All data in transit uses TLS 1.3.
How much does a Shopify API integration cost? +
Simple integrations (single endpoint, one-way sync) start from $500–$1,500. Complex bidirectional ERP or multi-system integrations with error handling, monitoring dashboards, and data transformation logic typically range from $3,000–$10,000+.
How long does a Shopify API integration take? +
A single-system integration such as Shopify to QuickBooks typically takes 1–2 weeks. Complex multi-system integrations connecting ERP, 3PL, and CRM simultaneously can take 4–8 weeks including sandbox testing, data validation, UAT, and staged production rollout.
Do you need Shopify Partner access to build integrations? +
For custom apps scoped to a single store, we work directly within your Shopify admin — no Partner access needed. For public apps intended for multiple stores or listing in the Shopify App Store, a Shopify Partner account and Partner Dashboard are required for app creation and review submission.
